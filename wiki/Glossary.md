# Glossary

Короткие определения терминов проекта. Подробности — по ссылкам.

- **RAG** (Retrieval-Augmented Generation) — поиск релевантных документов
  (резюме) перед подачей в LLM. Ядро проекта, обязателен по условию задачи.
  См. [[Архитектура]].
- **target** — бинарная метка пары вакансия×резюме: 1 = подходит на первое
  собеседование, 0 = нет. Ground truth для [[Евалюация]]. См. [[Датасет]].
- **Декартово произведение** — датасет содержит все пары `vacancy × resume`
  с разметкой, поэтому сущности повторяются → нужна дедупликация. См. [[Датасет]].
- **payload** — неэмбеддинговые поля в Qdrant для фильтрации (город, опыт,
  зарплата) без обращения к основной БД. См. [[Векторная_БД]].
- **Dense search** — поиск по эмбеддингам (семантика). См. [[Hybrid_Search]].
- **BM25** — лексический поиск по точным совпадениям термов. См. [[Hybrid_Search]].
- **RRF** (Reciprocal Rank Fusion) — объединение ранжированных списков dense и
  BM25. Поддержан Qdrant Query API (`prefetch` + `fusion=Fusion.RRF`).
- **Gap-анализ** — сопоставление профиля кандидата с требованиями вакансии,
  выявление отсутствующих требований. Шаг 2 в [[LLM_Pipeline]].
- **NDCG@k / MRR / Recall@k / Precision@k** — метрики качества ранжирования.
  См. [[Евалюация]].
- **instructor** — библиотека для structured output LLM через pydantic-схемы.
  См. [[LLM_Pipeline]].
- **ablation** — сравнение вариантов (моделей эмбеддингов, режимов поиска) для
  оценки вклада компонента. См. [[Евалюация]].

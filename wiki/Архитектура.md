# Архитектура

Общая схема системы **HR AI Scout**. Поток данных:
`data → vectorstore → retrieval → LLM → UI`, а параллельно — оффлайн-евалюация.

См. также: [[Index]], [[Датасет]], [[Векторная_БД]], [[Hybrid_Search]],
[[LLM_Pipeline]], [[Евалюация]], [[Roadmap]].

## Поток данных

```
total_df.csv
    │
    ├── Дедуп вакансий (3 409 уникальных) ──┐
    └── Дедуп резюме   (20 845 уникальных) ─┤
                                            ▼
                              Эмбеддинги вакансий / резюме
                                            │
                                            ▼
                      Векторная БД (Qdrant): collection resumes,
                                             collection vacancies
                                            │
                                            ▼
                   Retrieval: top-K резюме по вакансии
                   (hybrid: dense + BM25 + payload-фильтры)
                                            │
                                            ▼
                   LLM-слой: извлечение → gap-анализ → скоринг/объяснение
                                            │
                                            ▼
                              Streamlit UI

  Параллельно: eval/ — оффлайн-оценка retrieval на размеченных
  target=1/0 парах (NDCG@k, MRR, Recall@k, Precision@k)
```

## Слои и где про них читать

| Слой | Описание | Страница |
|---|---|---|
| Data | Чтение CSV, дедуп сущностей, сборка текста для эмбеддинга | [[Датасет]] |
| Vectorstore | Qdrant, две коллекции, payload, модель эмбеддингов | [[Векторная_БД]] |
| Retrieval | dense / BM25 / hybrid+RRF, фильтры по payload | [[Hybrid_Search]] |
| LLM | Цепочка из трёх промптов поверх top-K кандидатов | [[LLM_Pipeline]] |
| UI | Streamlit: выбор вакансии, список резюме, gap-анализ, метрики | (см. [[Roadmap]] Этап 6) |
| Eval | Оффлайн-метрики на ground truth `target` | [[Евалюация]] |

## Стек

- Векторная БД: **Qdrant** (Docker)
- Эмбеддинги: `cointegrated/LaBSE-en-ru` или `intfloat/multilingual-e5-large`
- LLM: через OpenAI-совместимый API (Claude/GPT-4o), `instructor` для structured output
- BM25: `rank_bm25` или Qdrant sparse vectors
- UI: Streamlit
- Данные: pandas, parquet
- Eval: собственная реализация метрик (без готовой RAG-eval библиотеки)

## Принципы скоупа

- Не переобучать classification-модель — `target` уже размечен.
- Индексировать **уникальные сущности**, а не все 332 330 строк (см. [[Датасет]]).
- RAG обязателен — нельзя засунуть обработку резюме в один LLM-вызов без поиска.

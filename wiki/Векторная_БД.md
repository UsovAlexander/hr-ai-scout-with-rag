# Векторная БД

Хранилище эмбеддингов проекта — **Qdrant** (локально через Docker).
Связано с [[Датасет]] (источник), [[Hybrid_Search]] (как ищем),
[[Архитектура]] (место в системе).

## Почему Qdrant

Бесплатно, легко поднимается, хороший Python-клиент, поддержка hybrid
search «из коробки» через Query API (`prefetch` + RRF) — см. [[Hybrid_Search]].

## Запуск

```bash
docker run -p 6333:6333 -p 6334:6334 \
  -v $(pwd)/qdrant_storage:/qdrant/storage \
  qdrant/qdrant
```

`docker-compose.yml` в корне для воспроизводимости:

```yaml
services:
  qdrant:
    image: qdrant/qdrant:latest
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - ./qdrant_storage:/qdrant/storage
```

## Две коллекции

Резюме и вакансии индексируются и ищутся **независимо** — это не одна
смешанная коллекция.

### `resumes`
- **vector:** эмбеддинг конкатенации `resume_title + resume_specialization +
  resume_last_position + resume_last_experience_description + resume_skills +
  resume_education + resume_courses`.
- **payload** (для фильтрации без похода в основную БД): `resume_id`,
  `resume_area = resume_location`, `resume_experience_months`, `resume_salary`,
  `resume_applicant_status`, `resume_age`.

### `vacancies`
- **vector:** эмбеддинг конкатенации `vacancy_name + vacancy_description +
  vacancy_experience`.
- **payload:** `vacancy_id`, `vacancy_area`, `vacancy_employment`,
  `vacancy_schedule`, `vacancy_salary_from`, `vacancy_salary_to`,
  `vacancy_salary_currency`.

Payload используется для фильтров при retrieval (город, опыт, статус поиска) —
реальный сценарий HR-скрининга, см. [[Hybrid_Search]].

## Модель эмбеддингов

Тексты на русском (резюме с hh.ru) → нужна модель под русский.

- `cointegrated/LaBSE-en-ru` — использовалась в предыдущих проектах.
- `intfloat/multilingual-e5-large` — сильнее на ретривале, чуть тяжелее.

Переключение модели вынести в конфиг (`src/config.py`) — нужно для ablation
в [[Евалюация]].

Эмбеддинги считаются батчами (`sentence-transformers`, `batch_size=64`,
GPU/MPS на Apple Silicon, иначе CPU). Это Этап 2 в [[Roadmap]].

## Open questions

- Для sparse-варианта BM25 (Qdrant sparse vectors) нужна отдельная
  конфигурация коллекции — см. [[Hybrid_Search]], решение не зафиксировано.
